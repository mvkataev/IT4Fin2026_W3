## Интеграция данных из нескольких источников: Мини-проект III.

В файле W3_16_Катаев.R содержится код для интеграции данных.

Здесь, в репозитории, хранятся два файла, которые получились в результате работы кода: лог-файл project_log.txt, в котором фиксируются ключевые действия и решения на всех этапах работы, и файл matching_report.csv с итоговым отчетом из Задания 8.

Важно, что в репозитории хранятся и файлы для загрузки данных: coins_latest.csv, cmc_latest (1).json (точная копия файла, приложенного к описанию заданий), cmc_map (1).json и w3_database.sqlite. В коде реализовано получение данных с помощью API CoinMarketCap (оттуда получены файл cmc_map (1).json и файл с info, используемый в заданиях 4-6, а также может быть получен файл cmc_latest (1).json). Однако я рекомендую скачать файлы именно отсюда, с репозитория, и не убирать комментарий из строчки 70, где может происходить загрузка. Для этого есть несколько причин. Во-первых, при вводе эндпоинта /v1/cryptocurrency/listings/latest данные могут скачаться не в полном объеме и быть меньше, чем данные из файла, приложенного к описанию заданий. Во-вторых, при парсинге данных для cmc_map сайт по некоторым ID иногда выдает некорректные данные. Например, при одном из запусков в поле ID у одной из криптовалют отображались просто экранированные символы, которые ломали процесс парсинга. В коде для работы с этой ситуацией реализован функционал, подсвечивающий именно ту часть данных, где происходит ошибка при парсинге. Поскольку ошибка бывает обычно только одна на весь файл и она действительно обычно связана с экранированными символами, ее можно исправить прямо вручную в файле json на основе подсказки из кода, это наиболее простое решение. Но лучше всего все же скачать именно тот файл, который есть в репозитории, равно как и все остальные файлы. С ними код работает гарантированно, это позволяет избегать наопределенности при парсинге данных.
